# Weather 天气助手 App 需求设计文档

## 1. 应用场景

### 1.1 目标用户
- 日常关注天气变化的普通用户
- 需要根据天气情况选择合适穿着的上班族
- 经常出差或旅行需要查看多个城市天气的用户
- 希望获得个性化穿衣和出行建议的用户

### 1.2 使用场景
- **日常出行**：上班、上学前查看当天天气和穿衣建议
- **旅行规划**：查看目的地未来几天天气，提前准备衣物
- **多城市关注**：同时关注家人、朋友所在城市的天气
- **智能问答**：通过AI助手获取个性化穿衣、出行、活动建议

### 1.3 核心价值
- 提供准确、及时的天气信息
- 智能推荐穿衣方案，减少用户决策成本
- AI助手提供个性化建议，提升用户体验
- 动态天气背景增强视觉体验

## 2. 功能需求

### 2.1 天气查询功能

#### 2.1.1 实时天气
- 显示当前城市名称、温度、天气状况
- 展示体感温度、湿度、风力风向等详细信息
- 天气图标动态显示

#### 2.1.2 天气预报
- 显示未来3天天气预报
- 包含日期、天气状况、最高温/最低温
- 横向滚动卡片展示

#### 2.1.3 定位功能
- 应用启动时自动获取当前位置
- 支持精确定位和粗略定位
- 定位失败时使用默认城市

### 2.2 城市管理功能

#### 2.2.1 城市搜索
- 支持城市名称搜索
- 显示搜索结果列表
- 点击添加到关注列表

#### 2.2.2 城市列表
- 显示已添加的所有城市
- 每个城市显示当前温度和天气
- 支持删除已添加城市

#### 2.2.3 城市切换
- 点击城市可切换首页显示
- 支持地图选点添加城市

### 2.3 穿衣推荐功能

#### 2.3.1 温度分级建议
| 温度范围 | 穿衣建议 |
|---------|---------|
| ≥ 30°C | 酷热，建议穿短袖短裤，注意防晒 |
| 25°C ~ 29°C | 温暖，短袖搭配薄裤会比较舒服 |
| 15°C ~ 24°C | 体感舒适，长袖或薄外套都合适 |
| 5°C ~ 14°C | 稍凉，建议加一件毛衣或夹克 |
| ≤ 4°C | 天气偏冷，羽绒服和保暖配件更稳妥 |

#### 2.3.2 天气状况修正
- 雨天：增加"建议携带雨伞"提示
- 大风：增加"注意防风"提示
- 雪天：增加"注意保暖防滑"提示

### 2.4 AI智能助手功能

#### 2.4.1 对话交互
- 支持文字输入提问
- 提供快捷问题选项
- 支持多轮对话

#### 2.4.2 智能回答
- 基于当前天气提供穿衣建议
- 根据天气推荐适合的活动
- 回答出行相关的问题

#### 2.4.3 上下文理解
- 记住当前城市天气信息
- 根据天气状况给出针对性建议

## 3. 界面需求

### 3.1 设计风格
- **主题**：Holo-Horizon（全息地平线）赛博流光极简视界
- **色彩**：深色背景 + 霓虹发光效果
- **字体**：无级变宽字体，数字采用金属光泽效果
- **图标**：3D线框全息投影风格

### 3.2 首页界面

#### 3.2.1 布局结构
- **顶部**：城市名称 + 管理城市按钮
- **中部**：当前温度（超大字体）+ 天气状况
- **详细信息**：湿度、风力、体感温度图标展示
- **穿衣建议**：3D线框图标 + 建议文字
- **底部**：未来3天天气预报卡片

#### 3.2.2 动态背景
- **晴天**：深邃暗色背景 + 流光太阳光效
- **雨天**：青蓝色粒子雨特效
- **雪天**：银白色3D粒子漂浮效果
- **阴天**：灰色调渐变背景
- **夜间**：深色星空背景

#### 3.2.3 交互效果
- 下拉刷新：全息蝴蝶/数据重载动效
- 陀螺仪视差：背景与前景元素分层移动
- 温度数字：金属光泽反光动画

### 3.3 城市管理界面

#### 3.3.1 搜索区域
- 命令行终端风格输入框
- 聚焦时闪烁光标效果

#### 3.3.2 城市列表
- 数据节点风格卡片
- 呼吸光效显示已关注城市
- 高对比度霓虹色调显示温度

### 3.4 AI助手界面

#### 4.4.1 弹出方式
- 底部弹层对话面板
- 半透明毛玻璃背景

#### 4.4.2 对话界面
- 消息气泡：用户蓝色，AI灰色
- 快捷问题芯片
- 加载动画效果

## 4. 技术要求

### 4.1 开发环境
| 项目 | 要求 |
|------|------|
| 开发语言 | Java |
| 最低SDK | Android 7.0 (API 24) |
| 目标SDK | Android 16 (API 36) |
| 构建工具 | Gradle + Kotlin DSL |
| IDE | Android Studio |

### 4.2 技术架构

#### 4.2.1 网络请求
- **Retrofit 2.9.0**：REST API客户端
- **OkHttp 4.12.0**：HTTP客户端
- **Gson 2.10.1**：JSON解析

#### 4.2.2 数据存储
- **Room 2.6.1**：本地数据库
- **SharedPreferences**：配置存储

#### 4.2.3 图片加载
- **Glide 4.16.0**：图片加载和缓存
- **Coil 2.7.0**：备用图片加载库

#### 4.2.4 UI组件
- **Material 3**：Material Design 3组件库
- **ConstraintLayout**：约束布局
- **RecyclerView**：列表展示
- **SwipeRefreshLayout**：下拉刷新

### 4.3 第三方API

#### 4.3.1 和风天气API
- **实时天气**：获取当前天气数据
- **天气预报**：获取未来3天预报
- **城市搜索**：搜索城市信息
- **逆地理编码**：经纬度转城市名称

#### 4.3.2 阿里云百炼AI API
- **智能体应用**：提供AI对话能力
- **OpenAI兼容接口**：标准化调用方式

### 4.4 权限要求
```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
```

### 4.5 性能要求
- **启动时间**：冷启动 ≤ 3秒
- **列表滑动**：帧率 ≥ 55fps
- **网络请求**：超时设置20秒
- **内存管理**：及时释放资源，避免内存泄漏

### 4.6 安全要求
- API密钥通过`local.properties`配置
- 禁止在代码中硬编码敏感信息
- 使用`BuildConfig`注入配置
- 网络请求使用HTTPS

### 4.7 兼容性要求
- 支持Android 7.0及以上版本
- 适配不同屏幕尺寸
- 支持深色模式
- 处理网络异常和定位失败情况

## 5. 数据模型

### 5.1 天气数据模型
```java
// 实时天气
WeatherResponse {
    Now {
        temp: String          // 温度
        feelsLike: String     // 体感温度
        icon: String          // 天气图标代码
        text: String          // 天气描述
        windDir: String       // 风向
        windScale: String     // 风力
        humidity: String      // 湿度
    }
}

// 天气预报
ForecastResponse {
    Daily {
        fxDate: String        // 日期
        tempMax: String       // 最高温
        tempMin: String       // 最低温
        iconDay: String       // 白天图标
        textDay: String       // 白天描述
    }
}
```

### 5.2 城市数据模型
```java
@Entity(tableName = "cities")
City {
    @PrimaryKey
    id: String           // 城市ID
    name: String         // 城市名称
    lat: double          // 纬度
    lon: double          // 经度
    addTime: long        // 添加时间
}
```

### 5.3 穿衣建议模型
```java
ClothingSuggestion {
    suggestion: String   // 建议文字
    iconResId: int       // 图标资源ID
    level: String        // 等级：hot/warm/cool/cold/freezing
    accessoryTip: String // 配件提示
}
```

### 5.4 AI对话模型
```java
AiChatMessage {
    role: String         // user/assistant
    content: String      // 消息内容
    loading: boolean     // 是否加载中
}
```

## 6. 模块划分

### 6.1 包结构
```
com.example.weather
├── MainActivity.java           // 首页
├── CityManageActivity.java     // 城市管理
├── MapPickActivity.java        // 地图选点
├── adapter/                    // 适配器
├── api/                        // API接口
├── model/                      // 数据模型
├── db/                         // 数据库
├── location/                   // 定位服务
├── service/                    // 业务逻辑
├── ui/                         // UI组件
└── utils/                      // 工具类
```

### 6.2 核心模块
1. **天气模块**：天气数据获取、缓存、展示
2. **定位模块**：位置获取、权限管理
3. **城市模块**：城市搜索、存储、管理
4. **穿衣模块**：穿衣建议生成
5. **AI模块**：AI对话、智能建议
6. **UI模块**：动态背景、图标管理

## 9. 风险与依赖

### 9.1 外部依赖
- 和风天气API服务稳定性
- 阿里云百炼AI服务可用性
- 用户设备定位功能

### 9.2 技术风险
- API调用频率限制
- 不同设备定位权限差异
- AI响应延迟

### 9.3 解决方案
- 实现数据缓存机制
- 提供默认城市备选
- 优化网络请求策略
- 添加加载状态提示
