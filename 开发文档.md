# 简易天气助手 + 穿衣推荐 App — 开发需求文档

## 1. 项目概述

### 1.1 项目名称
**Weather** — 简易天气助手 + 穿衣推荐

### 1.2 项目目标
开发一款 Android 天气应用，支持实时查询当地及多个城市的天气信息，并根据温度、天气状况智能推荐穿衣建议。界面简洁美观，背景随天气动态变化，并在首页集成 AI 智能天气助手，以对话形式提供更细致的穿衣、出行和活动建议。

### 1.3 目标用户
日常使用手机、关注天气和出行着装的普通用户。

### 1.4 开发环境与技术栈
| 项目 | 内容 |
|------|------|
| 开发语言 | Java |
| 最低 SDK | Android 7.0 (API 24) |
| 目标 SDK | Android 16 (API 36) |
| 构建工具 | Gradle + Kotlin DSL |
| UI 框架 | Material 3 (Material Design 3) + ConstraintLayout |
| 定位服务 | Android 系统 LocationManager |
| 网络请求 | OkHttp + Retrofit + Gson |
| 天气 API | 和风天气 (免费版) |
| AI 能力 | 阿里云大模型服务平台百炼智能体应用 API |
| 图片加载 | Glide (或 Coil) |
| 本地数据库 | Room (SQLite) |
| 下拉刷新 | SwipeRefreshLayout |

## 2. 功能需求

### 2.1 首页 — 天气展示

#### 2.1.1 当前天气大卡片
- 显示当前城市名称
- 实时温度（大号字体）
- 天气状况文字描述（如"晴"、"多云"、"小雨"）
- 天气对应图标（从和风天气 API 获取）
- 体感温度、湿度、风力信息
- 穿衣建议卡片

#### 2.1.2 未来天气预报
- 横向可滚动的 3-5 天天气预报卡片
- 每张卡片显示：日期/星期、天气图标、最高温/最低温

#### 2.1.3 动态天气背景
- 晴天：暖色调渐变 + 太阳光效
- 雨天：灰蓝色调 + 雨滴动效
- 雪天：冷色调 + 雪花飘落动效
- 阴天：灰色调渐变
- 夜间：深色背景

#### 2.1.4 下拉刷新
- 使用 SwipeRefreshLayout 实现下拉刷新
- 刷新时重新请求所有天气数据和定位

### 2.2 城市管理

#### 2.2.1 城市列表展示
- 显示用户已添加的所有关注城市
- 每个城市项显示：城市名、当前温度、天气状况

#### 2.2.2 添加城市
- 搜索框支持输入城市名称搜索
- 调用和风天气城市搜索 API
- 搜索结果列表展示，点击添加

#### 2.2.3 删除城市
- 长按或滑动删除已添加的城市
- 删除后首页对应城市数据移除

#### 2.2.4 数据持久化
- 使用 Room 数据库存储已添加的城市列表
- 应用重启后城市列表不丢失

### 2.3 穿衣推荐

#### 2.3.1 推荐逻辑
根据当日最高温/最低温/天气状况综合推荐：

| 温度范围 | 穿衣建议 |
|---------|---------|
| ≥ 30°C | 酷热，建议穿短袖短裤，注意防晒 |
| 25°C ~ 29°C | 温暖，建议穿短袖、薄裤 |
| 15°C ~ 24°C | 舒适，建议穿长袖或薄外套 |
| 5°C ~ 14°C | 较凉，建议穿毛衣或夹克 |
| ≤ 4°C | 寒冷，建议穿羽绒服、围巾手套 |

天气状况修正：
- 雨天 → 增加"建议携带雨伞"提示
- 大风 → 增加"注意防风"提示
- 雪天 → 增加"注意保暖防滑"提示

#### 2.3.2 展示形式
- 首页大卡片内嵌穿衣建议区域
- 文字描述 + 对应小图标（T恤、外套、羽绒服等）
- 不同建议使用不同配色

### 2.4 定位功能

- 应用启动时自动通过 LocationManager 获取当前位置
- 需要动态申请 `ACCESS_FINE_LOCATION` 和 `ACCESS_COARSE_LOCATION` 权限
- 定位成功后通过逆地理编码（和风天气 API）获取城市名称
- 定位失败时默认显示预设城市（如北京）

### 2.5 AI 智能助手

#### 2.5.1 入口与展示
- 首页右下角提供悬浮 `AI 助手` 入口按钮
- 点击后以底部弹层方式展开对话面板
- 面板包含标题、当前城市天气摘要、快捷提问 Chip、消息列表、输入框和发送按钮

#### 2.5.2 对话能力
- AI 助手基于当前选中城市的实时天气、3 日预报和穿衣建议生成回复
- 支持多轮对话，保留当前城市下的历史问答上下文
- 用户切换城市后，对话上下文自动切换到新城市天气
- 默认提供快捷提问：`今天怎么穿？`、`需要带伞吗？`、`适合什么活动？`

#### 2.5.3 智能体接入要求
- 接入阿里云大模型服务平台百炼的智能体应用 API
- 使用 OpenAI 兼容 Responses API 的同步调用方式
- API Key、App ID、Base URL 通过 `local.properties` 注入 `BuildConfig`，禁止硬编码
- 当 AI 服务不可用时，在聊天面板内给出明确错误提示，不影响天气主流程

## 3. 页面设计与交互流程

### 3.1 页面结构

```
App 启动
    │
    ├─ 权限检查（定位权限）
    │
    └─ 首页 (MainActivity)
        │
        ├─ 顶部：城市名 + 切换城市按钮 → 跳转城市管理页
        │
        ├─ 中部：当前天气大卡片
        │   ├─ 温度（大号）
        │   ├─ 天气状况文字 + 图标
        │   ├─ 湿度 / 风力 / 体感温度
        │   └─ 穿衣建议卡片（内嵌）
        │
        ├─ 下部：未来 3-5 天预报（横向滚动 RecyclerView）
        │
        ├─ 右下角：AI 助手悬浮按钮
        │
        └─ 底部：下拉刷新指示器
```

### 3.2 页面跳转

| 源页面 | 目标页面 | 触发方式 | 传递数据 |
|--------|---------|---------|---------|
| 首页 | 城市管理页 | 点击城市名/切换按钮 | — |
| 城市管理页 | 首页 | 选择城市 | 城市 ID / 城市名 |
| 城市管理页 | 首页 | 添加/删除城市后返回 | 更新后的城市列表 |

### 3.3 交互流程

```
启动 → 检查定位权限
  ├─ 已授权 → 获取定位 → 请求天气数据 → 展示首页
  └─ 未授权 → 动态申请
       ├─ 同意 → 获取定位 → 请求天气数据 → 展示首页
       └─ 拒绝 → 使用默认城市 → 请求天气数据 → 展示首页

下拉刷新 → 重新请求所有数据 → 更新 UI

点击城市名 → 跳转城市管理页
  ├─ 搜索城市 → 显示结果 → 点击添加 → 保存到 Room → 返回首页
  └─ 长按/滑动删除 → 从 Room 删除 → 更新列表

首页轮播城市天气 → 点击不同城市 → 更新首页天气数据

点击 AI 助手 → 展开底部对话面板
  ├─ 读取当前城市天气上下文 → 生成欢迎消息
  ├─ 点击快捷问题 / 手动输入 → 请求百炼智能体应用 API
  └─ 返回 AI 建议 → 更新聊天消息列表
```

## 4. 数据模型

### 4.1 天气数据模型 (来自和风天气 API)

```java
// 当前天气响应
public class WeatherResponse {
    private String code;
    private String updateTime;
    private Now now;      // 当前天气
    private List<Hourly> hourly; // 逐小时预报（可选）
    private List<Daily> daily;   // 逐日预报

    public class Now {
        private String temp;          // 温度，℃
        private String feelsLike;     // 体感温度，℃
        private String icon;          // 天气图标代码
        private String text;          // 天气状况描述
        private String windDir;       // 风向
        private String windScale;     // 风力等级
        private String humidity;      // 湿度，%
    }

    public class Daily {
        private String fxDate;        // 预报日期
        private String tempMax;       // 最高温
        private String tempMin;       // 最低温
        private String iconDay;       // 白天天气图标
        private String textDay;       // 白天天气描述
    }
}
```

## 5. API 接口设计

### 5.1 和风天气 API 配置

| 项目 | 内容 |
|------|------|
| 接口地址 | 使用和风天气项目专属 `API Host`，天气接口路径为 `/v7/...` |
| API Key | 需开发者自行在 [和风天气官网](https://dev.qweather.com) 注册获取 |
| 免费版限制 | 1000次/天调用，支持实时天气、3天预报、城市搜索 |

### 5.2 所需接口

#### 5.2.1 实时天气
```
GET /v7/weather/now
参数: location=城市ID 或 经纬度 & key=你的API_KEY
```

#### 5.2.2 3-5天天气预报
```
GET /v7/weather/3d  (免费版支持3天)
参数: location=城市ID 或 经纬度 & key=你的API_KEY
```

#### 5.2.3 城市搜索
```
GET /geo/v2/city/lookup
参数: location=城市名 & key=你的API_KEY
```

#### 5.2.4 逆地理编码 (通过经纬度获取城市)
```
GET /geo/v2/city/lookup
参数: location=经度,纬度 & key=你的API_KEY
```

### 5.3 阿里云百炼智能体 API 配置

| 项目 | 内容 |
|------|------|
| 接口地址 | `https://dashscope.aliyuncs.com/api/v2/apps/agent/{APP_ID}/compatible-mode/v1/responses` |
| API Key | 阿里云百炼 API Key |
| App ID | 百炼平台中创建的智能体应用 ID |
| 调用方式 | `Authorization: Bearer <API_KEY>`，同步 `POST` |
| 上下文 | 使用 `input` 传递 `system + 多轮 user/assistant` 消息数组 |

#### 5.3.1 智能助手请求示例

```json
{
  "input": [
    {
      "role": "system",
      "content": "你是 Weather 应用内的 AI 智能天气助手..."
    },
    {
      "role": "user",
      "content": "今天怎么穿？"
    }
  ],
  "stream": false
}
```

### 5.4 Retrofit 接口定义

```java
public interface WeatherApiService {
    @GET("v7/weather/now")
    Call<WeatherResponse> getCurrentWeather(
        @Query("location") String location,
        @Query("key") String apiKey
    );

    @GET("v7/weather/3d")
    Call<ForecastResponse> getForecast(
        @Query("location") String location,
        @Query("key") String apiKey
    );

    @GET("geo/v2/city/lookup")
    Call<CityResponse> searchCity(
        @Query("location") String cityName,
        @Query("key") String apiKey
    );
}
```

### 5.5 本地配置项

```properties
QWEATHER_API_KEY=你的和风天气Key
QWEATHER_API_HOST=你的和风天气API Host
ALIYUN_BAILIAN_API_KEY=你的阿里云百炼API Key
ALIYUN_BAILIAN_APP_ID=你的百炼智能体应用ID
ALIYUN_BAILIAN_BASE_URL=https://dashscope.aliyuncs.com
```

## 9. 权限清单

```xml
<!-- AndroidManifest.xml 需要声明的权限 -->
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
```

- `INTERNET`：网络请求和风天气 API
- `ACCESS_FINE_LOCATION` / `ACCESS_COARSE_LOCATION`：精确定位 / 粗略定位
- `ACCESS_NETWORK_STATE`：检查网络连接状态

## 10. 非功能性需求

- **性能**：首页冷启动 ≤ 3s（正常网络），列表滑动帧率 ≥ 55fps
- **安全**：和风天气 API Key、API Host 以及阿里云百炼 API Key、App ID、Base URL 不硬编码在代码中，统一通过 `BuildConfig` 或本地配置文件注入
- **兼容性**：支持 Android 7.0 (API 24) 及以上版本
- **可用性**：无网络时展示缓存数据并提示"网络异常"
- **可维护性**：遵循 MVC 分层，各模块职责清晰
- **AI 可靠性**：AI 助手异常时不阻塞主天气流程，失败信息仅在对话面板内反馈

## 11. 天气图标处理

和风天气 API 返回 `icon` 代码（如 `100` 代表晴），图标 URL 格式为：

```
https://developer.qweather.com/img/icon_weather/100.png
```

使用 Glide 加载图标到 ImageView，同时本地准备一套备用图标（drawable 资源）用于离线或无图模式。
