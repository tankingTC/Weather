# **Weather App UI/UX & 动效设计文档**

**设计主题：Holo-Horizon (全息地平线) —— 赛博流光极简视界**

## **1\. 核心设计理念**

* **极致去文字化 (Extreme Minimalism)：** 摒弃“温度：”、“湿度：”、“风力：”等冗余标签文本。完全通过直观的符号、色彩层级和巨大的数字产生视觉震撼。  
* **空间交互 (3D Spatial Interface)：** 打破 2D 平面限制。利用手机陀螺仪实现视差滚动（Parallax），让背景、卡片、数据层在 Z 轴上产生深度，营造第一人称的空间感。  
* **流光幻彩 (Iridescent Aesthetics)：** 核心 UI 元素采用“亮晶晶”的镭射/流光渐变材质，结合电子科幻风的暗色基底，突出光影流动的感觉。

## ---

**2\. 页面布局与视觉设计**

### **2.1 首页 (MainActivity) —— 全息数据看板**

首页是整个 App 的灵魂，采用全屏沉浸式设计，分为背景层、核心数据层和悬浮操作层。

* **动态天气背景层 (底层)：**  
  * **晴天：** 深邃暗色背景中心悬浮一个带有**流光溢彩（Iridescent）发光边缘**的太阳，光晕随时间缓慢呼吸。  
  * **雨天：** 采用类似《黑客帝国》**代码雨（Code Rain）变体的电子科幻粒子特效**。青蓝色的雨滴以发光数据流的形式快速垂直下落。  
  * **雪天：** 银白色的 3D 粒子在 Z 轴上缓慢漂浮，配合陀螺仪产生远近景交错的深度感。  
* **核心天气卡片 (中层)：**  
  * **布局：** 移除传统卡片的实体边框，使用极高透明度的“毛玻璃”（Glassmorphism）效果，边缘附带细微的霓虹发光线条。  
  * **温度展示：** 占据屏幕上半部分 1/3 的超大号、无级变宽字体（如 Roboto Flex）。数字本身带有金属光泽和反光动画，带来极强的视觉冲击力。  
  * **环境数据：** 在主温度下方，用一排极致简洁的发光小图标配合数字展示（💧 45% | 🍃 3级 | 🌡️ 26°）。  
* **穿衣推荐模块 (内嵌高亮区)：**  
  * **视觉呈现：** 摒弃传统的静态扁平图标。采用 **3D 线框全息投影（Hologram Wireframe）** 风格展示推荐衣物（如一件缓缓旋转的发光线框夹克）。  
  * **配色策略：** 根据温度等级采用不同颜色的霓虹发光：酷热（品红渐变）、舒适（青绿流光）、寒冷（冰蓝变幻）。  
* **未来天气预报 (下层)：**  
  * **交互：** 废弃传统的横向 RecyclerView。采用类似唱片机的 **3D CoverFlow 轮播效果**。当前聚焦的日期居中放大并点亮，两侧的日期在 3D 空间中向后倾斜并变暗。

### **2.2 城市管理页 (CityManageActivity) —— 卫星节点控制台**

* **整体风格：** 类似赛博朋克风的控制台面板，纯黑背景搭配高亮强调色。  
* **搜索框：** 设计成一个命令行终端输入框，聚焦时有闪烁的绿色/蓝色光标。  
* **城市列表：**  
  * 每个城市卡片是一个带有轻微呼吸光效的“数据节点”。  
  * 已关注城市的当前温度和天气图标，采用高对比度的霓虹色调（如赛博粉、电光蓝）显示。

**3\. App 核心动效设计 (Animations)**

在使用 Jetpack Compose 这样的现代声明式框架时，这些动效可以极其丝滑地实现。

### **3.1 下拉刷新：全息蝴蝶/数据重载**

* **放弃传统的转圈圈 Spinner。**  
* 用户下拉时，顶部会出现一个由光斑汇聚成的**流光蝴蝶（Butterfly Iridescence）或旋转的 3D 数据环**。  
* 松手刷新时，光环瞬间炸开（粒子消散特效），数据伴随着快速的数字跳动（类似老虎机滚动）刷新完成。

### **3.2 页面切换与入场动画**

* **启动 App：** 黑屏开场，一道高亮的光线从屏幕中央横向展开，随后界面元素如同“全息投影”一般从下至上、由暗到亮依次浮现（带有轻微的故障/Glitch 特效）。  
* **城市切换：** 点击切换城市时，当前城市的背景数据化为粒子消散，新城市的背景迅速在 3D 空间中重组。

### **3.3 陀螺仪视差联动 (Parallax)**

* 读取 Android 系统的陀螺仪传感器数据。  
* 当用户左右倾斜手机时，背景粒子（如雨滴、雪花）在底层移动缓慢，而顶层的温度数字和 3D 穿衣图标移动较快，形成极其逼真的空间悬浮感。

## ---

**4\. 技术实现建议 (基于你的技术栈)**

虽然你的需求文档写了使用 Java 和 Material 3，但考虑到你需要实现如此高维度的 UI 和动效，强烈建议在 UI 层引入 **Jetpack Compose**。

1. **3D 与粒子特效：** 使用 Compose 的 Canvas API 结合属性动画（animateFloatAsState）来绘制代码雨和流光背景。  
2. **流光材质 (Iridescent)：** 可以通过给 Modifier.background 传递一个带有动画 Offset 的 Brush.linearGradient 或 Brush.sweepGradient（包含紫、粉、青等多重色彩），配合 BlendMode 来实现光影流动的视觉。  
3. **空间视差：** 监听 SensorManager 的姿态数据，将计算出的 X/Y 偏移量应用到 Compose 的 Modifier.graphicsLayer { translationX \= ... } 上。
